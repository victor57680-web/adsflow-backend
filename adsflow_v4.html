☁️ Étape 1 — Créer un projet Google Cloud
1. Va sur console.cloud.google.com → connecte-toi avec le même compte Google que ton Google Ads MCC.

2. Crée un projet → nomme-le AdsFlow VDKustom

3. APIs & Services → Bibliothèque → cherche Google Ads API → Activer

4. APIs & Services → Identifiants → + Créer des identifiants → ID client OAuth 2.0
→ Type : Application de bureau (obligatoire pour générer un refresh token)
→ Nom : AdsFlow

5. Récupère le Client ID et Client Secret
🔑 Étape 2 — Developer Token Google Ads MCC
1. Va sur ads.google.com → connecte-toi sur ton compte MCC (administrateur)

2. Outils ⚙️ → Centre API

3. Clique Demander l'accès de base → remplis le formulaire

4. Attends 1–3 jours → tu reçois un email de confirmation

5. Retourne dans Centre API → copie ton Developer Token
⚙️ Étape 3 — Déployer le backend sur Railway
1. Va sur github.com → Nouveau repo → adsflow-backend
Upload : server.js, package.json, oauth.js
⚠ Ne pas uploader le .env

2. railway.app → New Project → Deploy from GitHub → sélectionne ton repo

3. Dans Railway → Variables → ajoute :
Copier
GOOGLE_CLIENT_ID=xxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-xxx
GOOGLE_DEVELOPER_TOKEN=xxx
GOOGLE_REFRESH_TOKEN=xxx (généré via node oauth.js)
GOOGLE_CUSTOMER_ID=tonIdSansTirets
4. Settings → Domains → génère un domaine → teste :
https://ton-url.railway.app/health → doit afficher {"status":"ok"}
